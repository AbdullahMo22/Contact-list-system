# نظام إدارة الاتصالات مع نظام الأدوار والصلاحيات (RBAC)

## وصف المشروع

نظام متكامل لإدارة الاتصالات مع نظام أدوار وصلاحيات متقدم، يتيح لكل مستخدم رؤية البيانات المخصصة له فقط بناءً على الفنادق والأقسام المرتبطة به.

## الميزات الرئيسية

### 1. نظام التوثيق والصلاحيات
- تسجيل دخول آمن باستخدام JWT
- نظام أدوار وصلاحيات متقدم (RBAC)
- فلترة البيانات حسب النطاق (الفنادق والأقسام)
- حماية جميع المسارات بالصلاحيات المطلوبة

### 2. إدارة المستخدمين
- إنشاء وتعديل وحذف المستخدمين
- تعيين أدوار متعددة للمستخدم
- ربط المستخدمين بالفنادق والأقسام
- تفعيل وإلغاء تفعيل الحسابات

### 3. إدارة الأدوار والصلاحيات
- إنشاء أدوار مخصصة
- تعيين صلاحيات للأدوار
- إدارة العلاقات بين الأدوار والصلاحيات
- واجهة سهلة لإدارة النظام

### 4. إدارة جهات الاتصال
- عرض جهات الاتصال المخصصة للمستخدم
- إضافة وتعديل وحذف جهات الاتصال
- فلترة البيانات حسب الفندق والقسم
- بحث متقدم في جهات الاتصال

### 5. تسجيل العمليات (Audit Log)
- تسجيل جميع العمليات مع IP وTime وMAC Address
- تتبع تعديلات المستخدمين والبيانات
- سجل شامل للأنشطة

## التثبيت والتشغيل

### 1. إعداد قاعدة البيانات
```sql
-- إنشاء قاعدة البيانات
CREATE DATABASE contact_management;
USE contact_management;

-- تشغيل سكريبت الجداول
SOURCE back/mysqlDataScript/Tables.sql;
```

### 2. تشغيل Backend
```bash
cd back
npm install
npm start
# Server will run on http://localhost:7000
```

### 3. تشغيل Frontend
```bash
cd front
npm install
npm run dev
# Frontend will run on http://localhost:5174
```

### 4. إنشاء البيانات الأساسية
```bash
cd back
node seeds/rbacSeed.js
```

## البنية التقنية

### Backend Framework
- **Express.js** - إطار العمل الخلفي
- **MySQL2** - قاعدة البيانات
- **JWT** - التوثيق
- **bcryptjs** - تشفير كلمات المرور
- **CORS** - التعامل مع Cross-Origin

### Frontend Framework
- **React 18** - مكتبة واجهة المستخدم
- **TypeScript** - لغة البرمجة المحسنة
- **Tailwind CSS** - إطار العمل للتصميم
- **Zustand** - إدارة الحالة
- **React Router DOM** - التنقل
- **Axios** - طلبات HTTP

### قاعدة البيانات
- **MySQL 8.0** مع الجداول التالية:
  - `users` - المستخدمين
  - `roles` - الأدوار
  - `permissions` - الصلاحيات
  - `user_roles` - علاقة المستخدمين والأدوار
  - `role_permissions` - علاقة الأدوار والصلاحيات
  - `hotels` - الفنادق
  - `departments` - الأقسام
  - `user_hotels` - ربط المستخدمين بالفنادق
  - `user_departments` - ربط المستخدمين بالأقسام
  - `contacts` - جهات الاتصال
  - `cards` - البطاقات
  - `audit_logs` - سجل العمليات

## API Endpoints

### Authentication
- `POST /api/auth/login` - تسجيل الدخول
- `GET /api/auth/me` - معلومات المستخدم الحالي

### Users Management
- `GET /api/users` - قائمة المستخدمين
- `POST /api/users` - إنشاء مستخدم جديد
- `PUT /api/users/:id` - تعديل مستخدم
- `DELETE /api/users/:id` - حذف مستخدم

### Roles Management
- `GET /api/roles` - قائمة الأدوار
- `POST /api/roles` - إنشاء دور جديد
- `PUT /api/roles/:id` - تعديل دور
- `DELETE /api/roles/:id` - حذف دور
- `POST /api/roles/:roleId/permissions/:permissionId` - تعيين صلاحية لدور

### Contacts Management
- `GET /api/contacts` - قائمة جهات الاتصال (مفلترة حسب النطاق)
- `POST /api/contacts` - إضافة جهة اتصال جديدة
- `PUT /api/contacts/:id` - تعديل جهة اتصال
- `DELETE /api/contacts/:id` - حذف جهة اتصال

## الأمان

### حماية الطرق
- جميع الطرق محمية بـ JWT Token
- فحص الصلاحيات لكل طريق
- فلترة البيانات حسب النطاق المسموح

### تشفير البيانات
- كلمات المرور مشفرة باستخدام bcrypt
- JWT Tokens آمنة ومحدودة المدة
- التحقق من الهوية في كل طلب

### تسجيل العمليات
- تسجيل جميع العمليات الحساسة
- تتبع IP Address و MAC Address
- سجل زمني لكل عملية

## الاستخدام

### 1. تسجيل الدخول
- استخدم أحد حسابات التجربة أو قم بإنشاء حساب جديد
- سيتم توجيهك للوحة التحكم

### 2. إدارة المستخدمين (للمديرين)
- انتقل إلى "إدارة المستخدمين"
- أضف مستخدمين جدد وعيّن لهم أدوار
- ربط المستخدمين بالفنادق والأقسام المناسبة

### 3. إدارة الأدوار والصلاحيات
- انتقل إلى "إدارة الأدوار"
- أنشئ أدوار جديدة وعيّن لها صلاحيات
- إدارة العلاقات بين الأدوار والصلاحيات

### 4. إدارة جهات الاتصال
- انتقل إلى "عرض الاتصالات"
- ستظهر لك جهات الاتصال المسموح لك برؤيتها فقط
- أضف أو عدل جهات اتصال جديدة حسب صلاحياتك

## الدعم والتطوير

### إضافة صلاحيات جديدة
1. أضف الصلاحية في جدول `permissions`
2. عدل الـ middleware للتحقق من الصلاحية الجديدة
3. حدث واجهة المستخدم لاستخدام الصلاحية

### إضافة مودولات جديدة
1. أنشئ النماذج (Models) المطلوبة
2. أضف الخدمات (Services) للمودول
3. أنشئ الـ Controllers والـ Routes
4. أضف واجهة المستخدم للمودول

## الأخطاء الشائعة وحلولها
